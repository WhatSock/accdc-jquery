Automatic Accessibility Framework

All AccDC Objects are accessible to screen reader and keyboard only users.

Key accessibility features:

Screen Reader Accessible Hidden Boundary Text
Screen reader accessible hidden boundary text conveys the beginning and ending boundaries of an AccDC Object to screen reader users.
ARIA Markup
ARIA markup applies an invisible heading structure to AccDC Objects to aid navigation for screen reader users.
Screen Reader Accessible Hidden Close Link
Screen reader accessible hidden close links ensure keyboard accessibility for all AccDC Objects.
Strategic DOM Insertion
Strategic DOM insertion automatically ensures a logical reading order for dynamically inserted AccDC Objects.
Strategic Focus Positioning
Strategic focus positioning forces keyboard focus to the beginning of AccDC Objects to aid navigation.
Screen Reader Accessible Hidden Role and State Text
Screen reader accessible hidden role and state text conveys dynamically updated role and state information within triggering elements to aid navigation.
Invisible Message Announcement
The AccDC Announce feature will announce the text of any string or DOM node to screen reader users, including automatic message stacking to prevent speech interruption when multiple messages are announced in rapid succession.

All AccDC Objects are compatible with the latest builds of leading Assistive Technology applications and supported browsers.
